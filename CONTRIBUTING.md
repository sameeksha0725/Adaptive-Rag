# Contributing to Adaptive RAG

Thank you for your interest in contributing to Adaptive RAG. We welcome small fixes, improvements, documentation, and tests.

Guidelines

- Fork the repository and create a feature branch: `git checkout -b feat/your-change`
- Keep changes focused and small; open a single PR per logical change.
- Run tests and `python -m compileall src` before submitting a PR.
- Follow project coding style in `CODE_STYLE_GUIDE.md`.
- Include tests for bug fixes and new behavior where practical.

Reporting Issues

- Use GitHub Issues to report bugs or request features.
- Provide steps to reproduce, expected vs actual behavior, and logs if available.

Code Review

- PRs will be reviewed for style, tests, and compatibility with the public API.
- Maintain backward compatibility unless the change is explicitly a breaking change and documented in the PR.

Thank you — maintainers
