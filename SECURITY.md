# Security Policy

## Supported Versions

The current repository state is considered the latest supported development branch.

## Reporting a Vulnerability

Please report suspected security vulnerabilities privately by opening a GitHub security advisory or contacting the maintainers directly. Do not disclose details publicly until a fix is available.

## General Guidance

- Keep secrets in environment variables or a secure secret manager.
- Do not commit .env files or API keys.
- Rotate exposed credentials immediately if a leak is suspected.
